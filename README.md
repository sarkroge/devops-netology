# devops-netology

Репозиторий для выполнения домашних заданий по курсу DevOps.

## Игнорируемые файлы

В каталоге `terraform` используется файл `.gitignore`, который содержит правила исключения файлов из репозитория.

Описание правил:

* `**/.terraform/*` — игнорируются все файлы и каталоги внутри директории `.terraform` на любом уровне вложенности.
* `*.tfstate` — игнорируются все файлы с расширением `.tfstate`.
* `*.tfstate.*` — игнорируются все файлы, имя которых начинается с `.tfstate.` (например, резервные копии состояния).
* `crash.log` — игнорируется файл с точным именем `crash.log`.
* `crash.*.log` — игнорируются все файлы, имя которых начинается с `crash.` и заканчивается на `.log`.
* `*.tfvars` — игнорируются все файлы с расширением `.tfvars`.
* `*.tfvars.json` — игнорируются все JSON-файлы переменных Terraform.
* `override.tf` и `override.tf.json` — игнорируются файлы с точными именами.
* `*_override.tf` и `*_override.tf.json` — игнорируются файлы, в имени которых присутствует `_override`.
* `.terraformrc` и `terraform.rc` — игнорируются конфигурационные файлы Terraform CLI.
* `*.tfplan` — игнорируются файлы плана выполнения Terraform.

## Обозначения в правилах

* `*` — обозначает любое количество любых символов.
* `**` — обозначает любое количество вложенных директорий.
