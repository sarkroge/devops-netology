# devops-netology

Репозиторий для домашних заданий по курсу DevOps.

## Игнорируемые файлы в Terraform

В каталоге `terraform` используется файл `.gitignore`, который содержит правила для исключения файлов из репозитория.

Описание правил:

- `**/.terraform/*` - игнорируются все файлы и папки внутри директории `.terraform` в любом месте проекта.
- `*.tfstate` - игнорируются все файлы с расширением `.tfstate`.
- `*.tfstate.*` - игнорируются все файлы, начинающиеся с `.tfstate.` (например, резервные копии состояния).
- `crash.log` - игнорируется файл с точным именем `crash.log`.
- `crash.*.log` - игнорируются все файлы, которые начинаются с `crash.` и заканчиваются на `.log`.
- `*.tfvars` - игнорируются все файлы с расширением `.tfvars`.
- `*.tfvars.json` - игнорируются все JSON-файлы переменных Terraform.
- `override.tf` и `override.tf.json` - игнорируются файлы с точными именами.
- `*_override.tf` и `*_override.tf.json` - игнорируются файлы, в имени которых есть `_override`.
- `.terraformrc` и `terraform.rc` - игнорируются конфигурационные файлы Terraform CLI.
- `*.tfplan` - игнорируются файлы плана выполнения Terraform.

Символ `*` в правилах означает любое количество любых символов.
Символ `**` означает любое количество вложенных директорий.
